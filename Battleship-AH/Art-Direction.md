# Battleship-AH Art Direction

## Visual Goal

Battleship-AH should feel:

- polished
- restrained
- tense
- modern
- premium
- readable at a glance

It should not feel:

- noisy
- toy-like
- cluttered
- overly military-sim
- overly cartoonish

## Temporary Execution Mode

The first implementation pass should use hollow wireframe visuals.

That means:

- outlined boards
- outlined ships or placement silhouettes
- outlined markers
- minimal fills
- strong color-coded state separation

The temporary mode should feel deliberate and clean, not like missing art.

## Core Visual Themes

The visual language should emphasize:

- hidden information
- station identity
- trust in the interface
- tension before impact
- satisfying reveal of hit, miss, and sink outcomes

## Surfaces That Need Design

### Platform Surfaces

- AlienHand logo and badge treatment
- station identity markers
- pairing screen layout
- endpoint/lobby presentation
- privacy curtain / pass-turn transition

### Game Surfaces

- player board treatment
- target board treatment
- ship placement tiles or silhouettes
- hit marker
- miss marker
- sunk feedback
- turn indicator
- victory / defeat screen

## Color Direction

Prefer a tight palette with strong separation between:

- private station state
- public/shared state
- actionable controls
- success/damage/result signals

The design should feel precise and confident rather than loud.

## Immediate Asset Priorities

1. logo and title treatment
2. board visual system
3. hit / miss / sunk markers
4. privacy transition screen
5. pairing screen
6. victory screen

These are the assets most likely to shape the whole product identity early.
