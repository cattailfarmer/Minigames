# Battleship-AH Sound Brief

## Audio Goal

Battleship-AH audio should feel:

- crisp
- elegant
- tactile
- calm but tense
- immediately informative

It should not feel:

- chaotic
- comedic
- over-produced
- harsh
- arcade-noisy

## Functional Cue List

### Platform And Lobby

- pairing accepted
- pairing rejected
- ready confirmed
- endpoint connected
- endpoint disconnected

### Match Flow

- ship placement confirm
- ship placement invalid
- turn start
- privacy curtain / handoff
- fire command
- hit
- miss
- sink
- victory
- defeat

## Design Rules

- each cue should have a distinct gameplay meaning
- no cue should be longer than it needs to be
- avoid layering too many sounds at once
- prioritize information clarity over spectacle

## Music Direction

If music is added in the first release, keep it minimal:

- lobby / pairing ambience
- low-intensity match bed
- stronger victory / defeat stinger

Music should support tension and polish without competing with state-change cues.
