# Battleship-AH Asset Plan

## Phase 1 - Prototype Assets

Used to prove flow, privacy, and state separation.

- hollow wireframe board grid
- hollow wireframe ship silhouettes or placement outlines
- simple wireframe markers for ship, hit, miss
- wireframe turn and transition screens
- placeholder button sounds

## Phase 2 - Production Identity Pack

Used to establish the product's public look and feel.

- title and logo
- board frame system
- marker icon set
- pairing screen visuals
- transition/privacy curtain visuals
- match result screens
- functional audio cue set

## Phase 3 - Steam Polish Pack

Used to make the release feel premium.

- refined UI motion
- victory and sink feedback polish
- endpoint connection/disconnection feedback
- trailer-quality screenshots and capture scenes

## Parallel Work Guidance

While implementation is still stabilizing, asset work should focus on:

- modular screen components
- icon families
- marker styles
- short functional sounds
- wireframe-first screen geometry

Avoid overfitting assets to exact code timing or layouts until the gameplay flow is stable.
