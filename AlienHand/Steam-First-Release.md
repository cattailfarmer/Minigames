# AlienHand Steam-First Release

## Release Position

AlienHand should not debut on Steam as an abstract platform promise.

It should debut through a polished flagship game that demonstrates the platform's value with minimal friction.

The first public release should therefore be:

- `Battleship-AH` as the playable product
- `AlienHand` as the platform identity and enabling technology

## Public Promise

The first Steam-facing promise must be clear and easy to believe:

- one host PC
- two private player stations
- hidden-state multiplayer
- LAN second-screen endpoint support
- polished setup and turn flow

Do not lead the first release with the full long-term platform vision.

Lead with the strongest present truth.

## What Must Feel Excellent

The first release must feel excellent in:

- setup clarity
- station pairing clarity
- privacy transitions
- per-player view readability
- turn flow
- overall polish and confidence

It does not need to prove every future endpoint and input configuration.

## What Must Be Deferred

The first Steam release should defer:

- Smart TV endpoint support
- raw multi-mouse support
- generalized multi-game platform UI
- experimental virtual-hand input systems
- large-scale endpoint/device combinatorics

Those are important to AlienHand's future, but they are not required for a strong debut.

## Internal Success Condition

The release succeeds if players can immediately understand:

- why AlienHand exists
- what makes Battleship-AH distinctive
- why one host with private stations is interesting

It fails if players are asked to admire architecture they cannot yet feel.
