# Core Delineation Skill

## Purpose (Waist)
Delineation is the cognitive act of *drawing precise boundary lines* around an entity to:
- Distinguish its interior from its exterior (division/separation)
- Trace its exact contour/outline (sketch/delimit)
- Attribute to it only the observable qualities that belong to its source identity (description with fidelity)

Etymology: Latin *dēlīneāre* ("to draw lines completely") — de- (completely) + līneāre (to draw a line).

## Workflow (3-Boundary Pants Structure)
1. **Specification (Waist — Identify Target)**  
   Name the raw entity in its undifferentiated form.

2. **Boundary Drawing (Leg 1 — Division)**  
   Trace the closed contour: explicitly state what is *inside* vs. *outside* the line.  
   Enforce Law of Non-Contradiction: no fuzzy overlap that violates identity in the same respect.

3. **Quality Attribution (Leg 2 — Description)**  
   List only the intrinsic, source-derived qualities that survive the boundary test.  
   Apply Paul-Elder standards: clarity, accuracy, precision, relevance, depth, fairness.
   - Perform identity recognition: first fully recognize and individuate the entity’s complete identity by enveloping its entire visible surface and outer contour (all observable boundaries, attributes, and context). Only then draw the delineation line and attribute qualities.
   - Normalize: Adjust the content, level of detail, language, and structure so that the output precisely matches the identified scope and context (e.g., brief and high-level for an index; detailed and operational for a dedicated skill file). This prevents mismatch, bloat, or insufficiency while maintaining relevance and scan efficiency.
   - Ensure the delineated entity is self-contained for its intended use scope: include minimal necessary definitions or summaries for any key terms, protocols, or assumptions so the output stands alone without requiring external documents. Apply this especially to modular components like skill files.
   - Identify the intended scope and context of the output. Maintain brevity and relevance for scan efficiency in large collections.

## Convergence (White Point)
The entity emerges cleanly individuated — ready for integration, comparison, or further decomposition without drift or bloat.

## Wide Application Examples
- **Instruction Sets**: Draw protocol boundaries (e.g., where one MetaCognition section ends and another begins).
- **Concepts**: Delimit "justice" (what it includes/excludes) and attribute essential qualities only.
- **Problems**: Set system boundaries before solving.
- **Communication**: Clarify terms to prevent equivocation.
- **Intuition**: Outline a nagging weight before full articulation.
