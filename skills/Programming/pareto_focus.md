# Skill: pareto-focus
name: pareto_focus
description: The skill of identifying the vital 20% of effort, code, features, or tasks that drive 80% of results, value, bugs, or impact, then ruthlessly prioritizing and focusing resources there while de-emphasizing the long tail.  
case: Activate when task volume overwhelms, during refactoring, feature prioritization, optimization, or when effort must yield disproportionate returns (e.g., bug fixing, MVP scoping, or learning a new technology).
file: pareto_focus.md

#### Purpose (Waist)
Pareto-focus is the skill of applying the 80/20 rule as a deliberate attention filter to maximize leverage in programming and development work.

#### Workflow (3-Boundary Pants Structure)

**Waist – Vital Few Declaration**  
Explicitly name the domain (codebase, backlog, features, learning goals) and declare the current state of overload or diffuse effort.

**Leg 1 – Identification & Ranking**  
Analyze data or patterns to surface the critical 20%: highest-frequency changes, most bugs, most user value, highest coupling, or core invariants. Use metrics, profiling, user feedback, or quick audits. Rank by impact.

**Leg 2 – Ruthless Prioritization & Action**  
Allocate primary resources to the vital 20%. De-scope or defer the remaining 80%. Re-evaluate after action to confirm leverage gained. Hand off results to Planner for path integration or Refiner for compression.

#### Core Rules (always enforce)
- Base identification on observable evidence or quick empirical checks, not assumption.  
- Ruthlessly cut or defer low-leverage items without sentiment.  
- Re-apply periodically; the 20% can shift.  
- Preserve epistemic-uncertainty on any ranking that lacks clear data.

#### Integration Rules
- Pairs with first-principles-decomposition (to find true core), inversion-premortem (to test leverage), and existing Refiner/Planner.  
- Feeds clean focus into Weaver for synergistic combinations and Scribe for recording high-impact anchors.

#### Boundaries / When Not to Use
- Do not use when uniform quality or exhaustive coverage is mandatory (e.g., safety-critical code requiring full verification).  
- Do not apply as excuse for incomplete work on agreed requirements.

#### Resolution
Diffuse effort collapses into high-leverage action. 80% of results emerge from disciplined focus on the vital 20%, dramatically improving efficiency and impact without proportional increase in work.
