# Skill: type1-type2_decision
name: type1-type2_decision
description: The skill of classifying decisions as irreversible (Type 1: slow, careful analysis) versus reversible (Type 2: fast, experimental) and adjusting deliberation speed and risk tolerance accordingly.  
case: Activate before architecture choices, library selections, deployment strategies, or any decision where reversibility affects tempo and safety.
file: type1-type2_decision.md

#### Purpose (Waist)
type1-type2_decision is the skill of treating decision reversibility as a primary filter to optimize speed, safety, and resource allocation in programming and system design.

#### Workflow (3-Boundary Pants Structure)

**Waist – Decision Classification**  
Explicitly name the decision and declare whether it is Type 1 (one-way door: high cost or impossible to reverse) or Type 2 (two-way door: easy and cheap to reverse).

**Leg 1 – Type 1 (Irreversible) Processing**  
Apply deep analysis: gather comprehensive data, run premortems, consult stakeholders, model long-term consequences. Move slowly and deliberately. Require high confidence before committing.

**Leg 2 – Type 2 (Reversible) Processing**  
Bias toward fast experimentation: prototype, A/B test, or implement minimally and monitor. Accept lower upfront certainty. Set explicit rollback or iteration triggers. Hand off outcome to Planner or feedback-loop-tuning.

#### Core Rules (always enforce)
- Classify based on actual cost/time of reversal, not perceived importance.  
- Never treat a Type 2 decision as Type 1 (avoids unnecessary paralysis).  
- Never treat a Type 1 decision as Type 2 (avoids reckless commitment).  
- Document the classification and rationale for future Scribe reference.  
- Preserve epistemic-uncertainty on borderline cases and re-classify if new information emerges.

#### Integration Rules
- Pairs with inversion-premortem (especially for Type 1), pareto-focus (prioritize analysis effort), and Security (protect against irreversible harm).  
- Feeds classified decisions directly into Planner for temporal path mapping and Weaver for resilient combinations.

#### Boundaries / When Not to Use
- Do not use for purely mechanical or trivial choices that carry no meaningful reversal cost.  
- Do not force every decision into rigid categories when context is highly fluid.

#### Resolution
Decisions are executed at the optimal speed and risk level. Type 1 choices gain necessary safety and depth; Type 2 choices gain speed and learning velocity. Overall tempo improves without sacrificing integrity.
